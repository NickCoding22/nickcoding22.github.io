Overview

In this assignment I implemented a mesh editing pipeline consisting of two main sections. In Section I, I implemented Bezier curve and surface evaluation using de Casteljau's algorithm: a recursive linear interpolation scheme that evaluates a curve or surface at a given parameter. In Section II, I worked with the half-edge mesh data structure to implement area-weighted vertex normals for Phong shading, local mesh operations (edge flip and edge split), and Loop subdivision for mesh upsampling. The half-edge representation made it straightforward to traverse neighborhoods and to update connectivity correctly when flipping or splitting edges.

Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm evaluates a Bezier curve at parameter t by repeated linear interpolation. Given control points and t, one step produces the next level of intermediate points: p'i = (1−t)pi + t pi+1. Applying this step recursively yields a single point on the curve.

I implemented this step in BezierCurve::evaluateStep(...): the function takes the current vector of 2D points and uses the class member t to compute and return the next level (one fewer point). I created a custom Bezier curve with 6 control points for the screenshots below.

Part 2: Bezier surfaces with separable 1D de Casteljau

For an n×n grid of control points and parameters (u, v), the separable algorithm evaluates each row at u to get one point per row, then evaluates that column at v to get the surface point. I implemented evaluateStep (one 3D de Casteljau step), evaluate1D (full 1D evaluation by repeated steps), and evaluate(u,v) (evaluate each row at u, then that column at v).

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

I computed the vertex normal as the normalized sum of (face area × face normal) over all incident non-boundary faces. Using HalfedgeCIter, I iterate around the vertex via h->twin()->next(), and for each face I get the three vertex positions, compute triangle area as half the cross-product length, and add area × face->normal(). Boundary faces are skipped.

Part 4: Edge flip

I implemented flipEdge by returning immediately for boundary edges, then collecting the two half-edges on the edge and the four spoke half-edges. I reassigned all six with setNeighbors so the edge connects the opposite vertices, and updated face and vertex half-edge pointers. No elements are created or deleted.

Part 5: Edge split

I implemented splitEdge by allocating one new vertex (at the midpoint), three new edges, two new faces, and six new half-edges, then wiring all pointers so the two triangles become four. The new vertex's half-edge points along the split edge. Boundary edges are skipped.

Part 6: Loop subdivision for mesh upsampling

I followed the recommended order: (1) Compute Vertex::newPosition for all vertices (Loop weight u = 3/(8n)) and mark vertices original. (2) Compute Edge::newPosition as 3/8(A+B)+1/8(C+D). (3) Split every original edge (from a saved list), set new vertex position and newPosition, and set isNew on vertices and edges. (4) Flip every new edge that connects an old and a new vertex. (5) Copy newPosition into position for all vertices. Sharp corners and edges round off; pre-splitting/pre-flipping can improve symmetry (e.g., on the cube).
